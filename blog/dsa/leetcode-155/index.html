LeetCode 刷題筆記 - 155. Min Stack

古古

2024/10/30

題目 #

LeetCode 連結

設計與實作一個 MinStack 的 class，能夠執行 push()、pop()、top()、getMin() 這 4 個操作

方法定義：

  • MinStack()：初始化此 stack object
  • void push(int val)：在 stack 中 push 一個元素 val（val 皆為整數）
  • void pop()：從 stack 中刪除最頂端的元素
  • int top()：返回 stack 中最頂端的元素
  • int getMin()：取得此 stack 中的最小值

上述的每一個方法的操作，時間複雜度必須為 O(1)

Example 1

Input
["MinStack","push","push","push","getMin","pop","top","getMin"]
[[],[-2],[0],[-3],[],[],[],[]]

Output
[null,null,null,null,-3,null,0,-2]

Explanation
MinStack minStack = new MinStack();
minStack.push(-2);
minStack.push(0);
minStack.push(-3);
minStack.getMin(); // return -3
minStack.pop();
minStack.top();    // return 0
minStack.getMin(); // return -2

解答 #

程式實作 #

  • 這題如果不需要設計 getMin() 方法，那就是超超超級簡單的 Stack 實作題，不如說整題會變成 Medium 難度，都是因為這個 getMin() 方法所導致的

    • 如果不需要實作 getMin() 方法的話，可以寫成下面這個樣子（就是一個最經典的 Stack）

      class MinStack:

    def __init__(self):
        self.stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)

    def pop(self) -> None:
        self.stack.pop()

    def top(self) -> int:
        return self.stack[-1]

  • 而要實作 getMin() 方法的話，有三種思考方式

    • 第一種是每一次 call getMin() 時，都去遍歷一遍 Stack 中的所有數字，這樣子的時間複雜度是 O(n)，不符合題目要求（題目要求所有操作都必須是 O(1)）
    • 第二種是每一次 call push() 時，都去計算一下當下的最小值是多少，然後儲存在一個變數裡面
      • 但是這樣子的做法仍舊無法滿足題目要求，因為假設那個最小值被 pop 掉之後，還是得用 O(n) 去遍歷整個 Stack 去更新目前的最新最小值
    • 第三種就是本題解法，也就是 「同步維護另一個 min_stack 的 Stack」，所以換句話說的話，就是同時間我們會有兩個 Stack 存在

      • 在這個解法中，當執行 push() 時，除了 push() 原始的 value 進到 stack 之外，也 push 另一個「當前最小值」的 value，進到 min_stack，所以當我們要查詢當前最小值時，只要回傳 min_stack 的最 top 值就好

      • 而假設執行 pop()，把 stack 中的原始值 -3 給刪掉時，這時候 min_stack 也執行一樣的 pop() 操作，把其對應的最小值刪掉，因此目前 min_stack 的 top 值，就是「stack 中剩餘的值的最小值」，因此這時候如果我們想要取得到最小值，一樣是只要返回 min_stack 的 top 就可以了

      • 所以透過 「同步維護另一個 min_stack 的 Stack」，就可以實作出 O(1) 的 getMin() 方法了！

  • 最終的解答

    class MinStack:

    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)

        # 更新 min_stack 中的值，注意此處要做 min_stack 為空的 edge case 判斷
        if not self.min_stack:
            self.min_stack.append(val)
        else:
            self.min_stack.append(min(val, self.min_stack[-1]))

    def pop(self) -> None:
        self.stack.pop()
        self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]

# Your MinStack object will be instantiated and called as such:
# obj = MinStack()
# obj.push(val)
# obj.pop()
# param_3 = obj.top()
# param_4 = obj.getMin()
