LeetCode 刷題筆記 - 242. Valid Anagram

古古

2024/10/30

題目 #

LeetCode 連結

給定兩個字串 s 和 t，判斷他們是否為 Anagram，如果為 Anagram 就 return true，否則 return false

Anagram 的定義：假設一個字串中的字母，可以被 rearranging（重新排序）成另一個字串，那麼這兩個字串即為 Anagram

Example 1

Input: s = "anagram", t = "nagaram"
Output: true

Example 2

Input: s = "rat", t = "car"
Output: false

解答 #

程式實作 #

  • 這題用兩組 Hash Map 就可以解決，概念就是分別記錄 s 和 t 這兩個字串中的所有字母出現次數，最後再比較一下他們必須相同，就可以了

    class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        # 先做個簡單的判斷，兩個長度不一樣就絕對是 False
        if len(s) != len(t):
            return False

        s_map, t_map = {}, {}

        # 分別記錄 s 和 t 的每個字母的數量
        for i in range(len(s)):
            if s[i] in s_map:
                s_map[s[i]] += 1
            else:
                s_map[s[i]] = 1

            if t[i] in t_map:
                t_map[t[i]] += 1
            else:
                t_map[t[i]] = 1

        # 比較 s_map 和 t_map 中的值是否完全一樣
        for key, value in s_map.items():
            if key in t_map and value == t_map[key]:
                pass
            else:
                return False

        return True

  • 這裡有一個地方可以小優化，就是 Python 中的 Hash Map 的 get() 寫法，可以給預設值，就可以優化上面把 s 和 t 轉換成 Hash Map 的過程

    class Solution:
    def isAnagram(self, s: str, t: str) -> bool:
        # 先做個簡單的判斷，兩個長度不一樣就絕對是 False
        if len(s) != len(t):
            return False

        s_map, t_map = {}, {}

        # 小優化，記錄 s 和 t 的每個字母的數量可以改寫成這樣
        for i in range(len(s)):
            s_map[s[i]] = s_map.get(s[i], 0) + 1
            t_map[t[i]] = t_map.get(t[i], 0) + 1

        # 比較 s_map 和 t_map 中的值是否完全一樣
        for key, value in s_map.items():
            if key in t_map and value == t_map[key]:
                pass
            else:
                return False

        return True

時間複雜度、空間複雜度分析 #

  • 時間複雜度： ${O(s + t)}$ ，s 為第一個字串的長度，t 為第二個字串的長度
  • 空間複雜度： ${O(s + t)}$ ，因為額外用到一組 Hash Map 來記錄

Follow-Up #

  • 這題可能會有一個 follow-up，就是面試官會問：「有沒有可能僅使用 O(1) 的空間複雜度來實作？」（因為上面的 Hash Map 的解法，空間複雜度為 O(s+t)）
    • 如果要實作空間複雜度為 O(1) 的解答，就是用 sort 排序來解
      • 因為這題的 Anagram 的特性是兩個字串的所有字母數量一樣，所以假設兩個字串排序過後，s 應該要等於 t 才對（否則就不是 Anagram 了）
      • 所以要達到空間複雜度為 O(1) 的話，就是用排序來解，但是時間複雜度就會漲到 O(nlogn)（因為排序的關係），所以雖然不是最佳解，但就是一個小 follow-up 這樣
