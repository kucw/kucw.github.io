工程師如何利用下班時間賺錢 - 6 個真實例子

By Kucw 2020/01/01 日常生活

我是一名 Java 後端工程師，平日我除了正常上班之外，下班和週末時間我曾嘗試過幾種工程師的方法來賺錢，以下分享給大家我的 6 個真實例子

PS: 我之前曾在大陸工作過一段時間，所以以下有些例子會跟大陸有相關

1. 做 Side Project - Line 掃碼點餐，結果：失敗

這是我回台灣第一個做的 side project，當初是覺得在大陸有很多進店掃桌上的 QR code 就可以點餐的機制，但是台灣卻沒有，所以想說可以把這一套搬到 Line 上來做。當時還特別在 Facebook、Linkedin 上發文，徵求合作夥伴，我是打算再找一個前端、一個 PM，然後加上我一個後端，組成一個三人的小團隊，分工合作完成各自的領域

想像很完美，現實總是很骨感的，到最後我只找到一個前端跟我一起做而已，跟我比較熟的幾個有想法的朋友，都不覺得這個東西會成功，其中有一個人提出來的原因是他覺得這個痛點不夠痛，對用戶來說，掃碼點餐能夠增進的效率可能不夠高到用戶會願意花時間去嘗試一種新的方式來點餐

雖然有不看好的聲音出現，我那時還是抱著我就想做這個題目的心情，跟那位前端一起繼續往下進行。我們討論了點餐流程、UI、後端如何串接…等規劃， 我也詢問我身邊有在餐廳工作的朋友（準確的說他是開餐廳），他們常用的點餐軟體是什麼？平常怎麼用的？問過之後我才發現了會導致這個 Line 點餐失敗的主要原因 — 它需要硬體設備的支援，而我們兩個根本拿不出這麼多錢去投資

事情是這樣的，當用戶使用 Line 點餐之後，他點餐的訂單會傳到店家的印表機，可能廚房會有一台，然後生食區也會有一台。光是要說服店家把舊有的系統換掉，改用新的點餐系統，就需要租或是賣給店家新的相容的印表機，而這個成本太高，並且投入的這些硬體設備，日後我也不知道該拿他們怎麼辦。還有一個是點餐系統後面有太多東西需要支援，像是進貨系統、報表系統、串接許多第三方支付（信用卡、Line Pay、Apple Pay、Google Pay….）等等，這些東西都是作為一個消費者看不到，但是作為一個店家需要考量到的地方

基於上面種種考量，所以這個 side project 最後以失敗告終，不過我從中學到了 Line Bot 如何開發，還學習到了像我這種什麼背景都沒有的工程師的 side project，最好是找純軟體相關的服務。像是寫一個 app 上架到 Apple Store 或 Google Play，這種的就滿好的，我要付出的只有我自己下班的私人休息時間來開發、或是頂多一兩台 Server 的機器錢而已，成本相對來說低很多，而且失敗了也不會怎麼樣

2. 兼職做外包程式，結果：失敗

我完全找不到有誰願意把東西發包給我做，感覺要能夠獨立接案的人需要有全端開發的能力，而我基本只打算接後端相關的（因為我想順便累積後端開發的實力），所以導致我完全接不到任何一個案子…

而且接案子可能還得有人脈吧，這方面我也沒有，所以接程式外包案賺錢在我身上完全失敗

3. 上 104 家教網接程式家教，結果：成功

以前對 104 的刻板印象就是只能教國英數物，沒想到現在也有人在徵程式家教了，而且徵的人還不少，以我的學歷（台大碩士資工所、交大資工系）來說，兩個月內能接到 4、5 個案子，每個案子的價格落在 600 ～ 1000。這種方式賺錢真的滿快的，我高峰的時候光家教費一個月賺了一萬多吧，相信能賺更多的大有人在，只是有的時候需要備課，會額外再多花一點時間

後來不接的原因是因為我覺得這樣用時間換錢太不值得了，我感覺人生的可能性還很多，應該把時間拿去探索其他領域上，提升自己更多的價值，而不是賺這個快錢。不過這個選擇見仁見智啦，也是有人能夠同時兼顧上班、家教、學習三部分，不過我自己嘗試後發現我不行，所以我就選擇了上班和學習這兩部分，這方面就看個人取捨囉

4. 做另一個 Side Project - 104 家教提醒助手，結果：半成功半失敗

這個 side project 算是意外做出來的，我當時很努力的想要接 104 的家教，希望當新的案子一出現時我就可以馬上知道該案子的詳細資訊，像是上課科目、上課地點、薪資待遇…等，而 104 家教網本身的 email 通知系統又太慢，它是一整天只會通知一次，當收到 email 再去看時通常已經被搶走了。所以我就做了個 Line Bot 爬蟲，每分鐘去看 104 家教網的網頁，比對有沒有新的案件出現，如果有新案件出現，就馬上發個 Line 通知給我

既然這個家教提醒助手 Line Bot 有成功做出來，而且也確實派上用場，為什麼我還會說這個結果是半成功半失敗呢？成功的點是我確實有把這個 side project 的功能完成，失敗的點是我曾經想要盈利這個 Line Bot，希望用戶每個月付給我多少錢，他就能使用這個助手，結果盈利失敗

後來想想，可能是我商業模式選得不好，不該採用訂閱制，願意為一個提醒助手買單的人比我想像中的少，所以才導致這個助手盈利失敗。嘗試賣這個助手失敗之後，我也曾經想過那乾脆就開放給所有人使用吧，結果受限於 Line api 的限制，Line Bot 每個月只能主動發送 500 筆訊息，只要超過就要收費。也因為這個限制，導致這個助手也不能讓大家一起用，所以這個助手在我不接 104 家教案子之後就有點荒廢了，算是有點可惜

5. 架個人網站試圖靠流量賺廣告費，結果：半成功半失敗

又是一個半成功半失敗的例子…，我曾經幻想著如果我架起了一個個人網站，然後在裡面寫許多文章，讓整個網站的流量上升，我就可以透過 Google Adsense 賺取流量廣告費，跟 Youtuber 在影片中安插廣告賺取廣告費的概念有點像

結果，事實是我個人網站確實架好了，文章也寫了不少（包含我以前在 CSDN 寫的技術文章），但是整體的流量卻沒有我想像中來得高，開張一個月只有 100 多個點閱率，慘的可以

探討了一下原因之後，發現是跟 Google 的 SEO 有關，因為我的網站流量太小，所以 Google 不會把我的網站排在搜尋引擎的前面，可能要翻到第三、第四頁才找得到我寫的文章，導致點閱率非常低。我猜測還有另一個原因，就是因為在中國無法查看 Github，而我的個人網站又架在 Github 上，所以我就賺不到中國網民的流量，才導致我的個人網站的瀏覽量比起我之前在 CSDN 上寫的文章少非常多（CSDN 是大陸的一個技術寫作平台）

6. 在 Hahow 線上平台開課，結果：待揭曉

Hahow 是台灣的一個線上付費學習的平台，在大陸這種線上付費學習的平台還滿常見的，我當初在大陸時也會線上買課來學。所以我就想說我是不是也可以來錄個線上課程擺在這個平台上，有興趣的人看到就會付費學習，也就是說我只要錄一次課，錢就會一直進來，我是這麼想像的

剛好我前面不是創建了一個個人網站嗎，我是使用 Github + Hugo 來創建這個個人網站的，我想說乾脆我就開一門線上課程，專門教人怎麼使用 Github + Hugo 來創建個人網站吧。所以我就選擇了這個主題，開始了 Hahow 的開課流程

Hahow 開課的流程還滿煩瑣的，首先你得先想好題目，然後想好每章節你大概要講什麼。設計課程大綱意外的難，因為你得想好哪裡要講什麼、或是哪個地方要準備實際例子來講解，等於你在設計課程大綱時，心中已經把整堂課程錄了一遍

寫完課程大綱之後，還得準備以下東西才能將課程送審

  • 封面圖
  • 課程介紹的文案，包含以下幾點
    • 你為什麼想開這門課程？
    • 你的課程核心價值在哪裡？
    • 課程單元怎麼規劃？
    • 課程教學方式？
    • 課程作業怎麼安排？
    • 什麼樣的人適合上這門課？
    • 上完這門課，學生可以學到哪些東西？
  • 1 ～ 3 分鐘的募資影片

上面那個募資影片真的快搞死我…，沒想到身為一個後端工程師，有一天我會打開 Premiere 去學怎麼剪片，感覺在 Hahow 上開課需要全能型的人才，個人專長、美術設計、寫作、拍影片的技能一樣不可少，各個領域的技能都要會一點，或是甚至要外包給別人來做，才有辦法把每一個部分做到最好

反正我是盡我所能把上面這些東西生出來了，目前我的課程正在送審中，就看 Hahow 那邊審下來會不會通過了，如果審核通過之後就要進行募資，當達到募資門檻之後，我就要開始錄製實際的課程啦，結果就等之後再揭曉囉
